Every tool has one markdown file that gets prepended to every chat: `CLAUDE.md`, `.github/copilot-instructions.md`, `.cursor/rules/project.mdc`, or `GEMINI.md`.

## What to put in it

**Do**
- Explain the project at a high level.
- Call out gotchas and things that aren't intuitive.
- Call out common terminology and key files.
- Include any common workflows.
- Update it when the AI is consistently getting something wrong.

**Don't**
- Don't put commands that are obvious to the AI (e.g. `npm run dev`, `git status`).
- Don't let it grow unbounded. When it does, break pieces out into skills, instructions, or rules.
- Don't let the AI draft the whole thing and ship it as-is. It'll add slop. This file should be concise and your own guidance.

Example first few lines of a `CLAUDE.md`:

```markdown
# my-app

Internal admin tool for the ops team to manage customer accounts, refunds,
and feature flags.

Stack: Next.js 15 + Postgres 16 + Drizzle. Auth through Clerk. Deployed on
Vercel. Playwright for e2e.

## Gotchas
- Migrations in `db/migrations/` are append-only. Never edit an applied one.
- `src/legacy/` is frozen. Don't touch unless asked.
- Auth paths always go through `requireUser()`. Never read cookies directly.

...
```

## Everything else

Each tool has a folder full of other surfaces: permissions, hooks, skills, subagents, commands. You don't need to memorize the paths. If you know the word, ask your AI to add it. "Add a hook that runs lint after every edit." Done.
