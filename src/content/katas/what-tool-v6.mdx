## The model matters less than the interface

Claude, GPT, Gemini -- they all write decent code. What actually changes your experience is the interface sitting between you and the model. The same developer, same codebase, same problem will work completely differently in a terminal versus a browser versus a background agent.

Right now there are five distinct ways to work with AI on code. Most developers don't know all five exist. This is about awareness, not ranking.

## 1. Terminal agents

A conversation in your shell. You prompt, the agent reads your repo, edits files, runs commands. Everything is visible. You're pair programming with someone who types faster than you.

**Tools**: Claude Code, Codex CLI, Gemini CLI, GitHub Copilot CLI

## 2. AI-native editors

Your editor with AI built in. Completions as you type, chat in a sidebar, agent modes for bigger changes. Your code is front and center. The AI assists without taking over.

**Tools**: VS Code + Copilot, Cursor, Antigravity, Windsurf

## 3. Browser builders

Open a URL. Describe what you want. Get a working app. No git clone, no node_modules, no local setup. Some include hosting and a database out of the box. The fastest path from idea to something running.

**Tools**: Replit, bolt.new, v0, Lovable

## 4. Agent dashboards

You manage work, not code. Describe tasks, spin up agents (sometimes several at once), review what comes back. The interface looks more like a project board than an editor.

**Tools**: Devin, Cursor Agents Window, Codex desktop app

## 5. Background agents

You don't interact in real time. An agent picks up a GitHub issue, plans an approach, writes code, runs tests, opens a PR. You review the result like you'd review a coworker's pull request.

**Tools**: GitHub Copilot coding agent, Claude Code background agents, Devin

## The boundaries are dissolving

These categories are already merging:

| Tool | Terminal | Editor | Dashboard | Background |
|------|----------|--------|-----------|------------|
| Claude Code | Yes | VS Code extension | -- | Yes (Cowork) |
| Cursor | -- | Yes | Yes (Agents Window) | -- |
| GitHub Copilot | Yes | Yes | -- | Yes (coding agent) |
| Devin | -- | -- | Yes | Yes |

A year from now this table will look different. The point isn't to memorize it. The point is to notice that you have options and that different work calls for different interfaces.
