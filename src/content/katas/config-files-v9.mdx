The one file that matters: `CLAUDE.md`, `.github/copilot-instructions.md`, `.cursor/rules/project.mdc`, or `GEMINI.md` (pick whichever your tool reads). It gets prepended to every chat.

## What goes in it

Only put things in here that the AI wouldn't already know or guess. A couple of lines on what the project is, then landmines and house rules.

```markdown
# my-app

Next.js 15 + Postgres. Playwright.

## Don't get these wrong
- Migrations are append-only. Never edit one that's applied.
- `src/legacy/` is frozen unless I say otherwise.
- Keep files under ~500 lines. Split when they grow.
- Prefer deleting code over adding flags.
```

Let the AI draft this for you: `/init` in your tool, or "read my repo and write a short CLAUDE.md."

## The rest of the surface

Below is the full config surface per tool. You don't have to memorize any of it. If you know the **word** (hook, skill, subagent, permission, MCP), ask your AI to add it and it'll figure out the paths.

```
your-project/
│
├── CLAUDE.md                          # always-loaded project instructions
├── .mcp.json                          # project MCP servers (committed)
├── .worktreeinclude                   # files to copy into git worktrees
└── .claude/
    ├── settings.json                  # permissions, hooks, env (team-shared)
    ├── settings.local.json            # personal overrides (gitignored)
    ├── rules/                         # split workspace rules (complement CLAUDE.md)
    ├── agents/
    │   └── reviewer.md                # custom subagent (Task tool can spawn)
    ├── commands/                      # /command shortcuts (older)
    ├── skills/                        # auto-invoked workflows (preferred over commands now)
    │   └── kata/SKILL.md
    ├── hooks/                         # shell scripts on tool events
    ├── output-styles/                 # response style presets (e.g. teaching.md)
    └── agent-memory/                  # Claude-written persistent agent memory
        └── <agent-name>/MEMORY.md

~/.claude.json                          # top-level user config (distinct from below)
~/.claude/                              # user-global, same shape as project
├── CLAUDE.md                          # personal preferences across all projects
├── settings.json                      # global permissions / deny lists
├── keybindings.json                   # custom keybindings
├── rules/, agents/, commands/, skills/, hooks/, output-styles/, agent-memory/
├── plugins/                           # installed marketplace plugins
└── projects/<project>/memory/MEMORY.md  # auto-memory per project
```

> **Note:** Commands and skills are now the same mechanism. For new workflows, use `skills/`.

A starter `.claude/settings.json`:

```json
{
  "permissions": {
    "allow": ["Bash(npm run *)", "Read", "Edit", "Write"],
    "deny": ["Bash(rm -rf *)"]
  }
}
```

A `.cursor/hooks.json`:

```json
{
  "version": 1,
  "hooks": {
    "beforeShellExecution": [
      { "command": "sh -lc 'block-dangerous-commands.sh'" }
    ],
    "afterFileEdit": [
      { "command": "sh -lc 'npm run lint:fix'" }
    ]
  }
}
```
