## Three ways to work with AI

Every AI coding tool falls somewhere across three categories. They're not rigid boxes. Most tools are blurring the lines, and most developers use more than one. But understanding the categories helps you pick a starting point.

**CLI** - The terminal is your interface. You type alongside the agent, it reads files, runs commands, edits code. You see everything it does. Best for developers who live in the terminal and want full control.

Examples: Claude Code, Gemini CLI, Codex CLI, Aider

**IDE** - Your code editor is the interface. AI lives in a sidebar, suggests completions, answers questions in chat, and can make edits across files. Your code is always front and center.

Examples: VS Code + Copilot, JetBrains + AI Assistant

**Agent-first app** - The agent dashboard is the interface. You give tasks, the agent works, you review results. Code editing is secondary to task management. Think of it as going from writing code to managing agents who write code.

Examples: Codex desktop, Cursor 3's Agents Window

## It's not one or the other

Most productive developers in 2026 use 2-3 tools. A common setup:

- IDE for daily editing and quick completions
- CLI for complex multi-file tasks and hard problems
- Agent-first for parallel tasks you want to run in the background

The tools are also converging. Cursor is an IDE that added an agent-first window. Claude Code is a CLI that has Cowork for background tasks. Copilot added agent mode. The categories describe where the tool started, not where it's stuck.

## Exercise

1. Ask your tool: "What kind of tool are you? CLI, IDE, or agent-first?"
2. Think about your last coding session. Were you mostly editing code, running terminal commands, or delegating tasks? That tells you which category fits your current workflow.
