## Start with how you work

Don't pick a tool based on hype. Pick based on what's in front of you when you're working.

| What's in front of you | Category | Examples |
|----------------------|----------|----------|
| A terminal | **CLI** | Claude Code, Gemini CLI, Codex CLI, Aider |
| A code editor | **IDE** | VS Code + Copilot, JetBrains + AI Assistant |
| An agent dashboard | **Agent-first app** | Codex desktop, Cursor 3 |

**CLI** tools are interactive. You and the agent work together in real time. You see every file it reads, every command it runs. Great for hard problems where you want to stay close to the action.

**IDE** tools meet you where you already are. Completions, inline chat, background suggestions. You keep editing, AI keeps helping. Lowest friction if you already have an editor you love.

**Agent-first apps** flip the model. You describe what you want, spin up agents, and review what comes back. Less hands-on, more delegation. Best when you have clear tasks you can hand off.

## The lines are blurring

These aren't rigid boxes. Cursor started as an IDE and just shipped an agent-first window. Claude Code is a CLI but also has Cowork for background tasks. Copilot added agent mode. Every tool is racing toward "all three."

Most developers in 2026 use 2-3 tools. 90% use at least one AI tool at work. The question isn't "which one" but "which combination."

## Exercise

1. What tool are you using right now? Ask it: "Are you a CLI, an IDE extension, or an agent-first app?"
2. Try a tool from a different category than your usual. If you're an IDE person, try a CLI agent for one task this week.
