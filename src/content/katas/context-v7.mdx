Think about what you'd do in each scenario. Then click to reveal.

<details>
<summary>Why is my session already using tokens before I've typed anything?</summary>

**Every tool preloads stuff for you.** Typical starting load is 15-40k tokens:

- **Config files**: `CLAUDE.md`, `AGENTS.md`, `.cursor/rules/*`, `GEMINI.md`.
- **Memory files**: user-level notes (`~/.claude/CLAUDE.md`, `~/.gemini/GEMINI.md`).
- **Tool definitions**: built-in tools plus every MCP server's tool schemas (often 5-20k tokens per MCP server).
- **System prompt**: the tool's own instructions.

Every MCP server you leave enabled is rent you pay on every session. Turn off what you don't need.
</details>

<details>
<summary>How do I see what's in my context right now, and how much I've used?</summary>

**Every tool has a way, usually one command or a hover.**

- **Claude Code**: `/context` breaks it down by category (system prompt, memory, tools, MCP, messages). `/status` shows the token count and model.
- **Cursor**: token indicator on the chat input shows current usage. Hover for a breakdown.
- **VS Code Copilot**: hover the token circle at the bottom right of the chat input.
- **Codex CLI**: `/status` shows session usage.
- **Gemini CLI**: `/stats` for usage, `/memory show` to see the concatenated memory.

![Claude Code /context output](/odd-ai-katas/images/katas/what-does-it-already-know/claude-code-see-context.png)

![VS Code Copilot context window showing token usage](/odd-ai-katas/images/katas/what-does-it-already-know/vscode-see-context.png)

Check it often, especially before a big task. Knowing you're at 60% full vs 20% changes whether you should clear or keep going.
</details>

<details>
<summary>Your AI just repeated a suggestion it already tried. What do you do?</summary>

**Clear and restart.** The context is stale and correcting it a third time just digs the hole deeper. Write a one-line summary of where you are, `/clear`, and paste the summary into a fresh session. Bigger context windows don't save you from this; Sonnet 4.5 retrieval drops to ~18% at 1M tokens. Past 30-40% full, you're in degraded territory.
</details>

<details>
<summary>You're mid-task, the window is getting heavy, but you need continuity. What do you do?</summary>

**Compact with a focus.** `/compact focus on the auth refactor, drop the test debugging` in Claude Code. `/compress` in Gemini CLI. This summarizes older turns into a short form so you can keep going. Do it on purpose at ~50-60% full, not when auto-compact kicks in at 80% with no steering.
</details>

<details>
<summary>You realize the AI took a wrong turn ten messages ago. What do you do?</summary>

**Rewind to that point.** `/rewind` or Esc+Esc in Claude Code opens a checkpoint picker. Three modes: restore conversation only, restore conversation + files, or keep the work and drop only the chat. Cursor has a per-message rewind button that undoes both chat and file edits. Checkpoints cover file-edit tool calls but not raw `rm`/`mv` in bash.
</details>

<details>
<summary>You want to try a risky approach without losing where you are. What do you do?</summary>

**Fork the session.** `/branch` in Claude Code, or `claude --resume <id> --fork-session`. The original thread stays untouched. Few tools have this natively. Files are shared across branches, so file writes still leak. Use it for "let me try this totally different thing and come back if it doesn't work."
</details>

<details>
<summary>You're wrapping up for the day and want to resume tomorrow. What do you do?</summary>

**Write a handoff file.** "Write HANDOFF.md: what you tried, what worked, what didn't, next steps." Tomorrow: start a fresh session with "Read HANDOFF.md and continue." Beats `claude --continue` into a long stale thread. Commits help too: a new session can read the last few commits to catch up cheaply.
</details>

<details>
<summary>The AI keeps making the same mistake across multiple sessions. What do you do?</summary>

**Get it out of chat and into the instructions file.** In Claude Code, start a message with `#` and it prompts where to save (project `CLAUDE.md`, user `~/.claude/CLAUDE.md`, or auto-memory). In Cursor, say "Turn this into a rule." In any tool: "Add this to CLAUDE.md." If you correct the same thing twice, the fix belongs in the file, not the conversation.
</details>

<details>
<summary>You're about to start a hard, multi-step task. What do you do?</summary>

**Plan, then reset.** Use plan mode (`/plan` or Shift+Tab in Claude Code, Plan tab in Cursor) to produce a concrete plan without writing code. Save it as `PLAN.md`. `/clear`. Start a fresh session that loads only the plan. A clean session with a precise plan beats a long session with accumulated corrections almost every time.
</details>

<details>
<summary>The AI needs context that isn't in your codebase (design docs, tickets, team discussion). What do you do?</summary>

**Pull it in through MCP.** Linear, GitHub, Notion, Confluence, Obsidian, and Slack all ship MCP servers in 2026. Connect the one you need and let the agent query it directly instead of you copy-pasting. Turn off MCP servers you're not using this session; each one spends 5-20k tokens on its tool schemas.
</details>

<details>
<summary>You open a fresh session and want to get caught up quickly. What do you do?</summary>

**Read recent commits.** "Read the last 5 commits and tell me where we left off." Cheaper than attaching a bunch of files, and the commit messages give shape to what's been happening. Works best when you commit small and often.
</details>

## Exercise

Pick the scenario you flubbed hardest and do it for real in your next session. If you didn't flub any, skim your last long conversation and find a moment where one of these moves would have saved you time.
