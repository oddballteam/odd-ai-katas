| Word | What it mean |
|------|-------------|
| **Token** | Tiny word-piece. Magic box eat these. One word is like one berry. Everything cost berries. |
| **Context window** | How much magic box can hold in head at once. When head full, it forget old things. Brain get smooshed. |
| **Model** | Which magic box brain you talking to. Some brain fast but dumb. Some brain slow but very smart. You pick. |
| **Prompt** | Everything you shove into magic box mouth. Not just your grunt, but cave rules and old conversation too. |
| **System prompt** | Secret instructions cave leader put in magic box BEFORE you talk. You no see, but it change everything. |
| **Skills** | Magic box not born knowing everything. Skill is like downloading kung fu into brain RIGHT WHEN NEEDED. You can tell it (`/review-pr`) or magic box hear key words in your grunt and grab right scroll on its own. |
| **Agent** | Magic box that DO things. Not just talk. It read your cave paintings, run your tests, change your code. Most tools now are agents. |
| **Subagent** | Scout you send out from cave. Does research in own head, not yours. Comes back with only the good stuff. Keeps your main cave brain clean and not full of junk. |
| **MCP** | Magic pipe that connect magic box to other caves. GitHub cave, Slack cave, database cave. Like secret tunnel between caves. |
| **Hooks** | Trip wire. When magic box walk through certain point, trip wire ALWAYS go off. No thinking. No choice. e.g. "ALWAYS check fire before leave cave." |
| **Commands / Slash commands** | Special grunts that start with `/`. Quick way to tell magic box do specific thing without big explanation. |
| **Rules / Instructions** | Cave wall paintings that tell magic box how tribe does things. e.g. "We use yarn, NEVER npm." Loaded every time. |
| **Plugins / Extensions / Customizations** | Gift basket you give magic box. Inside basket: skills, magic pipes, trip wires, special grunts. All bundled together. One basket might add Slack cave, deploy grunt, and safety trip wire. |
| **Human-in-the-loop** | Magic box stop and wait for you to grunt "yes" before doing thing. Sometimes good (check before big change). Sometimes annoying (asking permission for every little thing). Finding right balance is key. |
| **Dark code** | Code magic box write that NO HUMAN LOOK AT. You just say "yes yes yes" without reading. Bad. Invisible danger pile up. |
| **Agent orchestration** | Running MANY magic boxes at same time on different tasks. Like having whole tribe of magic boxes. |
| **Harness** | The body around magic box brain. Brain think, harness do. Claude Code, Cursor, VS Code are harnesses. Brain without harness just sit there. |
| **Temperature** | How wild magic box act. Low = same answer every time. High = creative, unpredictable. For code, keep low. |
| **Hallucination** | When magic box LIE with confidence. Say function exist that NOT EXIST. Make up cave that not real. Normal. Always check. |
| **RAG** | Magic box go READ REAL SCROLLS before answering, instead of just guessing from old memories. Much better. |
| **Context engineering** | Art of choosing WHAT to put in magic box head so it give good answer. The real skill. |
| **Memory** | What magic box remember between sleeps. Your preferences. Your cave layout. What made you angry last time. |

## Exercise

1. Pick word that sound useful. Grunt at tool: "Why would I use [word]? Give simple example."
2. Try: "Explain [word] like me caveman. No big words."
