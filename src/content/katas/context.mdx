Think about what you'd do in each scenario. Then click to reveal.

<details>
<summary>Why is my session already using tokens before I've typed anything?</summary>

**Every tool preloads stuff for you.** A bare session (no MCP, minimal config) runs roughly 5-25k tokens depending on the tool:

- Gemini CLI: ~6k
- Antigravity: ~10k+
- Claude Code: ~20k
- Codex CLI: ~21-22k
- Cursor, Windsurf, Copilot: not publicly measured, but leaked prompts suggest a similar band

Add MCP and big instruction files and it climbs fast:

- **Config files**: `CLAUDE.md`, `AGENTS.md`, `.cursor/rules/*`, `GEMINI.md`. Most people have 0 or 1 small file, which adds maybe 1-3k. A sprawling CLAUDE.md can add 10k+.
- **Memory files**: user-level notes (`~/.claude/CLAUDE.md`, `~/.gemini/GEMINI.md`). Usually small.
- **MCP servers**: 5-10k per server typical, 15-20k for heavy ones.

Every MCP server you leave enabled is rent you pay on every session. Turn off what you don't need.
</details>

<details>
<summary>How do I see what's in my context right now, and how much I've used?</summary>

**Every tool has a way, usually one command or a hover.**

- **Claude Code**: `/context` breaks it down by category (system prompt, memory, tools, MCP, messages). `/status` shows the token count and model.
- **Cursor / VS Code Copilot**: a circle indicator shows up near the chat input once a conversation starts. Hover it for the token breakdown.
- **Codex CLI**: `/status` shows session usage.
- **Gemini CLI**: `/stats` for usage, `/memory show` to see the concatenated memory.

Worth checking when the AI starts acting off. If the window is getting full, context bloat is probably the reason, and clearing will help more than any correction you type.
</details>

<details>
<summary>Your token usage is high and the AI is starting to miss things: forgetting earlier decisions, repeating itself, getting sloppier. What do you do?</summary>

**Clear and restart.** Model attention gets worse as context fills up, even well before the window limit. Summarize where you are in one line, clear, and paste the summary into a fresh session. (`/clear` in Claude Code and Gemini CLI, `/clear` or `/new` in Copilot CLI, "New Chat" in Cursor and VS Code Copilot.)
</details>

<details>
<summary>The AI keeps suggesting fixes that don't work, and you've corrected it twice. What do you do?</summary>

**Give it better inputs.** It's probably working from stale training data, vague symptoms, or a mental model of your system that's wrong. A few things that actually move the needle:

- Have it **add logs** and run again, then read the output.
- Point it at a **REPL** or let it run the failing command itself.
- **Paste a screenshot** of the error, UI glitch, or failing test output.
- Connect a relevant **MCP** (Sentry, Linear, your DB, Chrome DevTools).
- Tell it to **search the web** for the error + "2026"; library behavior changes fast.
- Give it the **official docs URL** or an up-to-date reference and tell it to read that specifically.
- Be more concrete: "what exactly are you trying and why do you think it will work this time?"

Repeating "still broken" without new inputs just burns tokens. New information unsticks it.
</details>

<details>
<summary>You're mid-task, the window is getting heavy, but you need continuity. What do you do?</summary>

**Compact with a focus.** `/compact focus on the auth refactor, drop the test debugging` in Claude Code. `/compact` in Copilot CLI. `/compress` in Gemini CLI. This summarizes older turns into a short form so you can keep going. Do it on purpose at ~50-60% full, not when auto-compact kicks in at 80% with no steering.
</details>

<details>
<summary>You realize the AI took a wrong turn ten messages ago. What do you do?</summary>

**Rewind to that point.** `/rewind` or Esc+Esc in Claude Code opens a checkpoint picker (conversation only, or conversation + files). In Cursor, VS Code Copilot, and Windsurf, each message has a restore button right next to it; click it to undo chat and file changes back to that point.
</details>

<details>
<summary>You want to try a risky approach without losing where you are. What do you do?</summary>

**Fork the session.** `/branch` in Claude Code, or `claude --resume <id> --fork-session`. The original thread stays untouched. Few tools have this natively. Files are shared across branches, so file writes still leak. Use it for "let me try this totally different thing and come back if it doesn't work."
</details>

<details>
<summary>You're wrapping up for the day and want to resume tomorrow. What do you do?</summary>

**Write a handoff file.** "Write HANDOFF.md: what you tried, what worked, what didn't, next steps." Tomorrow: start a fresh session with "Read HANDOFF.md and continue." Beats `claude --continue` into a long stale thread. Commits help too: a new session can read the last few commits to catch up cheaply.
</details>

<details>
<summary>The AI keeps making the same mistake across multiple sessions. What do you do?</summary>

**Get it out of chat and into the instructions file.** In Claude Code, start a message with `#` and it prompts where to save (project `CLAUDE.md`, user `~/.claude/CLAUDE.md`, or auto-memory). In Cursor, say "Turn this into a rule." In Copilot, "Add this to AGENTS.md" (or `.github/copilot-instructions.md`). If you correct the same thing twice, the fix belongs in the file, not the conversation.
</details>

<details>
<summary>You're about to start a hard, multi-step task. What do you do?</summary>

**Plan, then reset.** Use plan mode (`/plan` or Shift+Tab in Claude Code, Plan tab in Cursor) to produce a concrete plan without writing code. Save it as `PLAN.md`. `/clear`. Start a fresh session that loads only the plan. A clean session with a precise plan beats a long session with accumulated corrections almost every time.
</details>

<details>
<summary>The AI needs context that isn't in your codebase (design docs, tickets, team discussion). What do you do?</summary>

**Paste it, link it, or pull it in through MCP.** For a one-off, paste the relevant Slack thread, a screenshot, or a URL and ask the agent to fetch it. For repeat access, connect an MCP server if you can: Linear, GitHub, Notion, Confluence, Obsidian, and Slack all ship them in 2026. Heads up: locked-down company systems (Slack, Jira, internal wikis) often won't let you connect a third-party MCP server, so copy-paste is frequently the only option there. Turn off MCP servers you're not using this session; each one typically spends 5-10k tokens on its tool schemas.
</details>

<details>
<summary>You open a fresh session and want to get caught up quickly. What do you do?</summary>

**Read recent commits.** "Read the last 5 commits and tell me where we left off." Cheaper than attaching a bunch of files, and the commit messages give shape to what's been happening. Works best when you commit small and often.
</details>
