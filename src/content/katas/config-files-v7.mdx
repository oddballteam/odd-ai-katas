## Start with the markdown file

If you do nothing else, do this. **Every tool has a single project instructions file** that gets prepended to every chat and agent request: `CLAUDE.md`, `.github/copilot-instructions.md`, `.cursor/rules/project.mdc`, or `GEMINI.md`. Drop in your stack, your conventions, your commands. This is the lowest-effort, highest-value piece of config you can add. Skipping it is silly.

It's the foundation. You'll almost certainly want more on top (commands, hooks, skills) but everything else builds from here.

You don't have to write it from scratch. Each tool can scaffold one from your codebase:

- **Claude Code**: `/init`
- **GitHub Copilot**: `/init` in chat
- **Cursor**: `Cmd+Shift+P → New Cursor Rule` (or just ask the agent to generate one)
- **Antigravity**: ask the agent to create `GEMINI.md` from the repo

Then trim it down to what's actually load-bearing.

## When you outgrow it

Eventually you'll want more than one file: a permissions allowlist so you stop hitting "yes" on `npm test`, a saved `/ship` shortcut, a hook that runs `lint:fix` after every edit, a subagent that owns code review. Each tool has its own folder for that. Pick a tab to see the full surface.

## The shared pattern

No matter which tool, the same axis applies: **project files (committed) vs personal files (gitignored or in the tool's own settings store)**. Project = team-shared behavior. Personal = how *you* like to work, follows you across projects.

`AGENTS.md` is the cross-tool standard now read by Copilot, Cursor, Antigravity, Codex, and Aider. Claude Code still reads `CLAUDE.md` (and Antigravity also reads `GEMINI.md` as a higher-priority overlay).

## Exercise

1. Open one of your projects. Click through the tabs above and check which files actually exist for whichever tool(s) you use.
2. Pick one tool. Add the simplest missing file (an instructions file or an ignore file). See if the tool starts behaving differently.
3. Ask your AI: "which of my project's config files did you load at session start?" and verify the answer matches the relevant tab.
