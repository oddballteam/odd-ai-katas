## The landscape is shifting

A year ago, "AI coding tool" meant autocomplete in your editor. Now there are terminals that edit your code, IDEs that run agents, desktop apps that spin up parallel tasks, browser tools that build full apps from a prompt, and autonomous agents that work while you sleep.

This isn't about picking the best one. It's about knowing what exists so you can match the right interface to the work in front of you.

## Terminal agents

You type a prompt. The agent reads your files, edits code, runs commands, and reports back. Everything happens in your terminal. You see what it sees.

**Tools**: Claude Code, Codex CLI, Gemini CLI, GitHub Copilot CLI

## AI-integrated IDEs

Your editor, but with AI woven in. Completions as you type, a chat sidebar for questions, and agent modes that can plan and execute multi-file changes in the background. Your code stays front and center.

**Tools**: VS Code + Copilot, Cursor, Antigravity, Windsurf

## Browser-based builders

You describe what you want in a browser. The tool generates a working app, sometimes with hosting, a database, and auth included. No local setup required. Good for prototypes, MVPs, and people who don't want to touch a terminal.

**Tools**: Replit, bolt.new, v0, Lovable

## Autonomous agents

You file an issue or describe a task. An agent picks it up, plans an approach, writes code, runs tests, and opens a PR. You review the output, not the process. Some run in the cloud, some run locally in the background.

**Tools**: Devin, GitHub Copilot coding agent, Claude Code (background agents)

## The lines are blurring

Most tools don't fit neatly into one box anymore:
- Cursor has an IDE and an agent window
- Claude Code runs interactively or as a background agent
- GitHub Copilot works in the CLI, the IDE, and the browser
- Windsurf was acquired by Cognition (who makes Devin)

You don't need to pick one category. Most developers use 2-3 tools. The question isn't "which is best" but "which interface fits this task right now."
