| Term | What you need to know |
|------|----------------------|
| **Token** | The unit AI reads and writes in. ~3/4 of a word. Everything costs tokens. A 200-line file is about 800. |
| **Context window** | The AI's memory limit for one session. Everything has to fit: your messages, its responses, files, instructions. When full, older content gets compacted or summarized. |
| **Model** | The specific AI brain (Opus, Sonnet, GPT-4). Different speed, cost, and smarts. You can usually switch. |
| **Prompt** | Everything sent to the model. Not just your message, but system instructions, files, and history bundled together. |
| **System prompt** | Hidden instructions your tool sends before you type anything. Your CLAUDE.md or copilot-instructions.md lives here. |
| **Harness** | The software that wraps the AI model and gives it tools, permissions, and rules. Claude Code, Cursor, and VS Code Copilot are all harnesses. The model is the brain, the harness is the body. |
| **Agent** | AI that takes actions: reads files, runs tests, edits code, creates commits. Most AI coding tools are agents. |
| **Subagent** | A child agent spawned by the main agent to handle a specific subtask. e.g. the main agent kicks off a subagent to research a library while it keeps coding. |
| **MCP** | Model Context Protocol. Plug-in system connecting your AI to external services. e.g. an MCP server that lets your AI read and create GitHub issues. |
| **Skills** | Portable packages of instructions and scripts that extend what your AI agent can do. e.g. a skill that knows how to review PRs in your team's style. |
| **Hooks** | Automated actions that fire at specific points in the agent's workflow. Unlike skills, hooks run outside the AI's reasoning. They enforce rules the AI can't skip. e.g. "always run lint before committing." |
| **Commands / Slash commands** | Shortcuts built into your tool (like `/context`, `/init`, `/memory`). Increasingly these are just skills you can invoke by name. |
| **Rules / Instructions** | Configuration files that tell the AI how to behave in your project. e.g. "always use yarn, never npm" in your CLAUDE.md. Loaded every session automatically. |
| **Plugins** | Extensions that add capabilities to your tool. MCP servers are the modern version. Older tools used proprietary plugin formats. |
| **Human-in-the-loop** | A workflow where the AI proposes changes but a human approves before they're applied. The default for most coding agents. The alternative is full autonomy. |
| **Dark code** | Code generated by AI that no human has read or understood. Builds up when you accept suggestions without reviewing. Creates invisible tech debt. |
| **Agent orchestration** | Running multiple agents in parallel on different tasks. The Codex desktop app and Cursor 3 are built around this concept. |
| **Temperature** | Randomness dial. Low = deterministic, predictable code. High = creative, varied output. Coding tools keep it low. |
| **Hallucination** | AI confidently stating something false. e.g. referencing a function that doesn't exist or inventing an API flag. Normal. Always verify. |
| **RAG** | Retrieval-Augmented Generation. AI searches real documents before answering instead of guessing from training data. |
| **Context engineering** | Deliberately shaping what information the AI sees so it gives better results. The real skill in using AI tools well. |
| **Memory** | What your tool remembers between sessions. Your preferences, project context, past corrections. |
